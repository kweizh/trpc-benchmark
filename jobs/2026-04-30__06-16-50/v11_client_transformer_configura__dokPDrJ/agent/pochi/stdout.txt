## Fix Applied: tRPC v11 Transformer Configuration

The `superjson` transformer has been moved from the root `createTRPCClient` options into the `httpBatchLink` configuration, which is the correct placement for tRPC v11.

### What changed in `client/trpc.ts`

**Before (v10 pattern — broken in v11):**
```ts
export const trpc = createTRPCClient<AppRouter>({
  transformer: superjson,              // ❌ root level — not valid in v11
  links: [
    httpBatchLink({
      url: 'http://localhost:3000',
    }),
  ],
});
```

**After (v11 pattern):**
```ts
export const trpc = createTRPCClient<AppRouter>({
  links: [
    httpBatchLink({
      url: 'http://localhost:3000',
      transformer: superjson,           // ✅ inside the link options
    }),
  ],
});
```

### Why this matters

In tRPC v11, the `transformer` property was removed from `CreateTRPCClientOptions` and moved into individual link options (`httpBatchLink`, `httpLink`, `wsLink`). Keeping it at the root level would silently not apply serialization, causing `Date` objects (like the `2026-04-28T10:00:00.000Z` returned by `getServerTime`) and other complex types to fail deserialization on the client.

The server-side `initTRPC.create({ transformer: superjson })` in `server/router.ts` was left untouched as required.
