### Key Implementation Details

1. **tRPC Server** (`lib/trpc/routers/upload.ts`):
   - Uses `zfd.formData({ file: zfd.file() })` for server-side validation
   - Reads the file buffer via `file.arrayBuffer()` and saves to `public/uploads/`
   - Generates timestamp-prefixed filenames to avoid collisions
   - Returns `{ success, url, filename, originalName, size }`

2. **Client Component** (`app/_components/FileUpload.tsx`):
   - Uses `trpc.upload.uploadFile.useMutation()` with `@tanstack/react-query` v5
   - Passes native `FormData` directly to the mutation — no serialization needed
   - Displays loading spinner, error messages, and the uploaded image preview

3. **API Route** (`app/api/trpc/[trpc]/route.ts`):
   - Standard tRPC v11 `fetchRequestHandler` pattern for Next.js App Router
   - Exports both GET and POST handlers
