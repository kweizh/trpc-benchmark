## Key Implementation Details

1. **tRPC Router** (`src/server/trpc.ts`): Uses `initTRPC.create()` with `superjson` transformer. Defines `addMessage` mutation with zod input validation (`z.object({ text: z.string() })`), returning `{ success: true, message: text }`.

2. **createCallerFactory**: In tRPC v11, `createCallerFactory` is accessed via `t.createCallerFactory` (returned from `initTRPC.create()`), not as a direct import from `@trpc/server`. This was a breaking change from v10.

3. **Server Action** (`src/app/actions.ts`): A `"use server"` function that creates a new caller instance and invokes `caller.addMessage({ text })` directly — no HTTP roundtrip needed.

4. **Client Component** (`src/app/MessageForm.tsx`): Uses `useTransition` for pending state, client-side validation for empty input, and displays success/error messages from the server response.
