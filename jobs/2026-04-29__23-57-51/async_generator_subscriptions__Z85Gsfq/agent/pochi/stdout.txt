### What was built

**`/home/user/project/server.ts`**  
A Node.js HTTP server using tRPC v11's `nodeHTTPRequestHandler`. The `countdown` procedure uses an `async function*` (AsyncGenerator) as the subscription handler, accepted natively by tRPC v11's `.subscription()` API:

```ts
countdown: t.procedure
  .input(z.number().int().min(0))
  .subscription(async function* ({ input }) {
    for (let i = input; i >= 0; i--) {
      yield i;
      if (i > 0) await new Promise(resolve => setTimeout(resolve, 100));
    }
  })
```

The server manually strips the `/trpc/` URL prefix to derive the bare procedure path, which is the expected format for `nodeHTTPRequestHandler`'s `path` option.

**`/home/user/project/client.ts`**  
A tRPC client using `httpSubscriptionLink` (SSE transport). Because Node.js v24 doesn't expose `EventSource` globally, the `eventsource` npm package is used as a ponyfill — both passed explicitly to the link option and registered on `globalThis` (required for tRPC's internal SSE readyState checks).
