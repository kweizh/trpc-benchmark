## What was built

### Architecture
- **`server.ts`** — tRPC v11 router with a `chatStream` **query** procedure that returns an `AsyncGenerator<string>`, yielding `"hello"`, `"world"`, and `"!"` with 100ms delays. Served via `createHTTPServer` (standalone adapter) on port 3000.
- **`client.ts`** — Creates a typed tRPC client using `httpBatchStreamLink` pointed at `http://localhost:3000`. Calls `client.chatStream.query()`, which resolves to an `AsyncIterable<string>`, then iterates it with `for await...of`, logging each chunk. Output is saved to `client.log`.
- **`tsconfig.json`** — TypeScript config targeting ES2022 with CommonJS modules.

### Key design decision
`httpBatchStreamLink` supports **streaming query responses** (procedures returning `AsyncGenerator`/`AsyncIterable`) but **not** subscriptions (those require `httpSubscriptionLink` or `wsLink`). The `chatStream` procedure is therefore a `.query()` that returns an `AsyncGenerator`, which the server serializes as a streamed HTTP response and the client receives as an `AsyncIterable`.
