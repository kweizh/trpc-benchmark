Fixed `client/trpc.ts` for tRPC v11 by moving the `superjson` transformer from the root `createTRPCClient` config into the `httpBatchLink` options object.

**The change (`client/trpc.ts`):**

```ts
// Before (broken v10 style):
export const trpc = createTRPCClient<AppRouter>({
  transformer: superjson,          // ❌ root-level — removed in v11
  links: [
    httpBatchLink({ url: 'http://localhost:3000' }),
  ],
});

// After (correct v11 style):
export const trpc = createTRPCClient<AppRouter>({
  links: [
    httpBatchLink({
      url: 'http://localhost:3000',
      transformer: superjson,      // ✅ per-link — required in v11
    }),
  ],
});
```

**Why:** tRPC v11 removed the top-level `transformer` option from `createTRPCClient`. Each terminating link (`httpBatchLink`, `httpLink`, `wsLink`, etc.) must now carry its own `transformer` option so the serializer is applied at the network boundary.
