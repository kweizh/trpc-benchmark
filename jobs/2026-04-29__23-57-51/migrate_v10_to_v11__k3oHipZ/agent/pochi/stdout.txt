### `src/utils/trpc.ts` — transformer placement
In v11 the transformer can no longer live at the root of the `config()` return object. It must appear in two places:
1. **Top-level** on `createTRPCNext({})` options — used by the SSR hydration layer to deserialize state.
2. **Inside `httpBatchLink`** — used for wire-level serialization.

```ts
export const trpc = createTRPCNext<AppRouter>({
  transformer: superjson,            // ← v11: top-level for SSR hydration
  config(opts) {
    return {
      links: [
        httpBatchLink({
          url: `${getBaseUrl()}/api/trpc`,
          transformer: superjson,    // ← v11: inside the link for wire encoding
        }),
      ],
    };
  },
  ssr: false,
});
```

### `src/server/context.ts` — `TRPCRequestInfo` / `getRawInput()`
In v11, `createContext` no longer receives procedure input directly. The `opts` object now carries an `info: TRPCRequestInfo` field. Raw input must be fetched asynchronously via `info.calls[0].getRawInput()`:

```ts
export async function createContext(opts: trpcNext.CreateNextContextOptions) {
  const rawInput = opts.info.calls.length > 0
    ? await opts.info.calls[0].getRawInput()
    : undefined;

  return {
    req: opts.req,
    res: opts.res,
    rawInput,
  };
}
```
