### Key implementation details

**Server — `chatStream` AsyncGenerator procedure** (`server/routers/_app.ts`):
```ts
chatStream: publicProcedure
  .input(z.string().min(1).max(500))
  .query(async function* ({ input }): AsyncGenerator<string> {
    for (const word of words) {
      yield word + " ";
      await new Promise(r => setTimeout(r, 40 + Math.random() * 80));
    }
  })
```
tRPC v11 natively detects `async function*` and streams each `yield`ed value over HTTP.

**Client — `httpBatchStreamLink`** (`utils/trpc-client.ts`):
```ts
export const trpcClient = createTRPCClient<AppRouter>({
  links: [httpBatchStreamLink({ url: "/api/trpc" })],
});
```
`httpBatchStreamLink` (not `httpBatchLink`) reads the chunked HTTP response incrementally — this is the v11 feature that enables streaming without WebSockets.

**Client consumption** (`app/page.tsx`):
```ts
const stream = await trpcClient.chatStream.query(input);
for await (const chunk of stream) {
  // append each chunk to the UI in real time
}
```
The returned async iterable is iterated with `for await...of`, and each chunk updates the React state immediately, producing a live typewriter effect.

**API route** (`app/api/trpc/[trpc]/route.ts`): Uses `fetchRequestHandler` from `@trpc/server/adapters/fetch` — compatible with Next.js App Router's runtime.
