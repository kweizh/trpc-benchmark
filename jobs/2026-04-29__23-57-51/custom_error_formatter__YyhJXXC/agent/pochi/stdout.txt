The custom `errorFormatter` has been added to `initTRPC.create()` in `/home/user/app/server.ts`. Here's a summary of the changes:

**1. Added `errorFormatter` to `initTRPC.create()`:**
```ts
export const t = initTRPC.create({
  errorFormatter({ shape, error }) {
    return {
      ...shape,
      data: {
        ...shape.data,
        zodError:
          error.code === 'BAD_REQUEST' && error.cause instanceof ZodError
            ? error.cause.flatten()
            : null,
      },
    };
  },
});
```

**2. Fixed a pre-existing Express v5 type compatibility issue** — split the chained `app.use(express.json(), '/trpc', ...)` into two separate `app.use()` calls, which is the correct pattern for Express v5.

**How the formatter works:**
- It spreads the default `shape` and `shape.data` to preserve all existing error fields.
- It checks `error.code === 'BAD_REQUEST'` **and** `error.cause instanceof ZodError`.
- If both conditions are true (i.e. a Zod validation failure), `zodError` is set to `error.cause.flatten()`, which produces a structured `{ formErrors: [], fieldErrors: { title: [...] } }` object.
- For all other errors, `zodError` is `null`.
